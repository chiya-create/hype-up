## シナリオ 1: 未ログイン時

**目的:** 保護ページへのアクセスが正しく制限されることを確認

- [ ] `https://your-domain.com/` → トップページが表示される（アップロードフォームが見える）
- [ ] `https://your-domain.com/projects` → 「ログインが必要です」メッセージが表示される
- [ ] `https://your-domain.com/admin` → 「アクセスできません」（ShieldAlert）が表示される
- [ ] `https://your-domain.com/onboarding` → `/login` にリダイレクトされる
- [ ] `https://your-domain.com/compare` → ログインしていない場合の状態を確認

## シナリオ 2: platform_admin ログイン

**目的:** platform_admin が全機能にアクセスできることを確認

### ログイン

- [ ] `/login` にアクセス → platform_admin メールを入力 → 「ログインリンクを送信」
- [ ] メールが届く（受信トレイまたは迷惑メールフォルダを確認）
- [ ] リンクをクリック → `/auth/callback` を経由してリダイレクト
- [ ] トップページのヘッダーにメールアドレスと「Platform Admin」が表示される

### Admin 画面

- [ ] ヘッダーの `Admin` リンク → `/admin` が表示される
- [ ] KPI カードが表示される（組織数・プロジェクト数等）
- [ ] 「組織管理」リンク → `/admin/organizations` が開く
- [ ] 「利用ログ」リンク → `/admin/usage` が開く
- [ ] 「集計インサイト」リンク → `/admin/insights` が開く
- [ ] 「フィードバック」リンク → `/admin/feedback` が開く

### 組織管理

- [ ] `/admin/organizations` で組織一覧が表示される
- [ ] 「組織を作成」フォームから新規組織を作成 → 一覧に追加される
- [ ] 作成した組織の「詳細」→ `/admin/organizations/[id]` が開く
- [ ] 「メンバーを招待」フォームにテストメールを入力 → 「招待」ボタンが動作する

### アクセス制限

## シナリオ 4: CSV アップロード

**目的:** ファイルアップロードとプロジェクト作成が正常に動作することを確認

- [ ] トップページのフォームでプロジェクト名を入力（例: 「スモークテスト - コスメ」）
- [ ] 業界テンプレートを選択（例: 美容・コスメ）
- [ ] サンプル CSV（`sample-reviews-cosme.csv`）をアップロード
- [ ] アップロード完了後にプロジェクト詳細ページ（`/projects/[id]`）に遷移する
- [ ] プロジェクトのステータスが `pending`（分析待ち）と表示される
- [ ] レビュー数が 30 と表示される（サンプル CSV の件数）
- [ ] `/projects` 一覧に作成したプロジェクトが表示される

---

## シナリオ 4.5: CSV 件数上限エラー

**目的:** 1,000 件超の CSV を拒否することを確認

- [ ] 1,001 行以上（ヘッダー除く）の CSV ファイルを用意する
  - 既存 CSV をコピーして行を増やすか、`yes "review text" | head -1001` 等で生成
- [ ] フォームに CSV を選択する
  - 期待値: ファイル選択後に「件数超過」エラーが赤色で表示される
  - 期待値: 「アップロードして分析準備」ボタンが **無効化（グレーアウト）** される
- [ ] （フォームを迂回して直接 API を叩く場合）`/api/upload` に 1,001 件の CSV を POST
  - 期待値: HTTP **400** + `{ error: "現在の MVP では 1 プロジェクトあたり最大 1,000 件..." }` が返る
- [ ] 500 件超 1,000 件以下の CSV を選択する
  - 期待値: 「件数注意: ○○件の分析には数分かかる場合があります」の amber 警告が表示される
  - 期待値: ボタンは有効（送信可能）

## シナリオ 5: AI 分析実行

**目的:** Claude API を使った分析が正常に完了することを確認

- [ ] プロジェクト詳細ページで「AI 分析を開始」ボタンをクリック
- [ ] ステータスが `analyzing`（分析中）に変わる
- [ ] 1〜3 分後にステータスが `done`（完了）に変わる
  - ステータスが変わらない場合: ページをリロードして確認
  - `error` になった場合: `/admin/usage` でイベントログを確認
- [ ] 分析完了後、分析結果セクションが表示される

---

## シナリオ 10: 組織分離（RLS 確認）

**目的:** RLS によるデータ分離が正しく機能していることを確認

### 準備

- Org A: client_owner_a@example.com（テスト済みのユーザー）
- Org B: 別の組織に属す client_owner_b のプロジェクト ID を用意

### テスト

- [ ] client_owner_a としてログイン
- [ ] Org B のプロジェクト URL（`/projects/[org_b_project_id]`）を直接入力
  - 期待値: **404 Not Found** が表示される
- [ ] Org B のプロジェクトに対する API アクセス（`/api/projects/[org_b_project_id]/feedback`）
  - 期待値: **403 Forbidden** または空レスポンス
- [ ] `/projects` 一覧に Org B のプロジェクトが**表示されない**ことを確認
