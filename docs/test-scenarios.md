# 検証シナリオ一覧

品質検証の手順書です。各シナリオを実施後、`docs/quality-check-guide.md` のチェックリストで評価し、問題があれば `docs/prompt-improvement-log.md` に記録してください。

---

## 前提

- サンプル CSV は `/public/samples/` に用意済みです
  - `sample-reviews-cosme.csv` — 美容・コスメ（30件）
  - `sample-reviews-education.csv` — 英語スクール・教育（30件）
  - `sample-reviews-hotel.csv` — ホテル・観光（30件）
  - **注意**: いずれも架空のレビューデータです。実際の商品・サービス・人物とは一切関係ありません。
    source 列の値（amazon / rakuten / google など）も架空のプラットフォーム表記です。
- 検証環境: `http://localhost:3000`（本番環境の場合はURLを変更）

## 検証後の対応

1. チェックリスト（`docs/quality-check-guide.md`）で NG があったセクションをメモ
2. `docs/prompt-improvement-log.md` に問題の詳細と改善方針を記録
3. 改善後に該当シナリオを再実行して変化を確認
